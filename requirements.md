/superpowers:brainstorming
Claude Code용 웹 풀스택용 대형 하네스를 하나 설계하려고 한다. (플러그인으로도 진화할 수 있는 형태로)
어떤 프로젝트인지 정의가 되면, 이를 기반으로 맞춤 디자인 시스템, 백엔드 필요 여부 등 서비스의 큰 틀을 구축하고, 스킬을 통해서 기획 인터뷰를 진행하면 자동으로
필요한 화면, 기능, 동작, 요구사항 등을 기획하고, 프론트엔드와 백엔드간 API 규약을 맞추고, 백엔드 개발하고 프론트엔드를 개발하는 그런 워크플로우를 제공하는
플러그인이다.

러프한 요구사항은 아래와 같다.

- 프로젝트의 초기화 단계가 있다. 이 단계에서는 이 프로젝트의 목적, 사용자 등등 실제 스타트업에서 신규 서비스를 기획하듯 사용자와 아주 면밀하게 인터뷰를
  진행하고, 이를 기반으로 디자인 시스템을 비롯하여 프론트엔드 프로젝트 초기화, 백엔드 프로젝트 초기화 등의 이 서비스의 전체적인 큰 틀?을 확정하고 구현하는 프로젝트 세팅 스킬이 필요하다.
  - 프로젝트 초기화 단계에 대해서는 실제 성공한 스타트업에서 신규 프로젝트를 빌드하는 과정을 면밀하게 리서치하여 차안할 것을 나에게 제안한다.
- 프론트엔드 프레임워크(기술 스펙)는 이 플러그인에서 React + NextJS + ShadCN + Tailwind CSS 조합으로 개발한다.
  - 이에 대해서 React + NextJS + ShadCN + Tailwind CSS를 사용하는 최신 프론트엔드 개발 방법론에 대해서 리서치하고, 이를 이 하네스(플러그인)에 반영 
- 백엔드 프레임워크(기술 스펙)는 Spring + Kotlin + KoTest Behavior Spec + DDD 기반의 조합으로 개발한다.
  - 이에 대해서 Spring + Kotlin + KoTest Behavior Spec + DDD를 사용하는 최신 백엔드 개발 방법론에 대해서 리서치하고, 이를 이 하네스(플러그인)에 반영
- 인프라의 경우에는 최대한 무료를 사용한다. (프론트엔드 배포 = vercel, 백엔드 = Oracle cloud, DB = Supabase)
- 인프라의 경우 사용자가 직접 가입하고 등록해야 하는 것들은 프로젝트 세팅 스킬을 실행하고 구현한 후 가이드를 만들어야 하며, 이후 인프라 세팅은 IaC(Terraform)을 활용하여 사용자가 인프라를 직접 구성하지 않고 코드로 관리할 수 있도록 한다.

위 요구사항을 만족하는 하네스 기획을 더 고도화시키기 위해 브레인스토밍 진행한다.
브레인스토밍을 진행하기 위해, 최신 AI 하네스 설계 이론(특히 Claude Code), 신규 프로젝트를 기획하고 발전시켜나가는 과정 등을 이미 검증되고 많이 사용되는 이론으로만 면밀하게 리서치하여 최선의 방법으로 구현하는 하네스를 만들 수 있도록 한다.
섣불리 추론하지 않는다. 조금이라도 애매한 지점이나, 모호한 지점에 대해서는 면밀하고 세심하게 나와 인터뷰 진행하여 기획의 완성도를 높인다.
신규 프로젝트를 만들려는게 아니다. 앞으로 신규 프로젝트들을 만들 때 적용할 수 있는 웹 풀스택용 하네스를 설계하기 위해 브레인스토밍하는 것이다. 
Claude Code용 하네스(플러그인)를 설계하기 위한 기획을 구체화 하는 것에 목적을 두어라.
